# 作品集：Nuomi Circuit - 多模态 AI 角色扮演伴侣

## 1. 项目概述 (Project Overview)
- **项目名称**：Nuomi Circuit
- **我的角色**：全栈开发者 / 产品设计 (独立负责系统架构设计、后端核心开发与产品体验规划)
- **项目周期**：2025年1月 - 2025年2月
- **项目简介**：一款基于 Telegram 的高性能多模态 AI 伴侣系统。该项目深度整合了大语言模型（GLM-4.7）、流式语音识别（ASR）、高保真语音合成（TTS）以及图像生成（GLM-Image）技术，突破了传统纯文本聊天的限制，为用户提供沉浸式的全双工角色扮演体验。

## 2. 用户调研发现 (User Research Findings)
在项目立项初期，通过对现有 AI 陪伴类产品及开源社区（如 SillyTavern）用户的调研与需求分析，我发现了以下核心痛点：
1. **交互维度单一**：纯文本聊天缺乏情绪张力和沉浸感，用户渴望能“听到”角色的声音并感知情绪的起伏。
2. **长文本记忆遗忘**：随着对话轮数增加，API Token 极易超限，导致 AI 经常遗忘关键的早期设定或剧情，破坏角色扮演的连贯性。
3. **视觉反馈缺失**：在剧情推进到特定场景时，缺乏贴合上下文的视觉画面（如场景照片）来增强代入感。
4. **平台门槛高**：专业的角色扮演工具（如 SillyTavern）配置繁琐、网络要求高，普通用户很难在移动端随时随地获得高质量的体验。

## 3. 设计与开发过程 (Design & Development Process)
针对上述痛点，我主导了系统的从 0 到 1 的设计与开发，主要分为四个关键阶段：

### 阶段一：移动端无缝接入与交互设计
- **平台选择**：选择 Telegram 作为前端交互入口，利用其原生支持的长轮询和多媒体（语音、图片）能力，免去用户下载独立 App 的烦恼。
- **交互状态机**：设计了清晰的用户交互流（欢迎 -> 角色选择 -> 沉浸式对话），支持 `/start`, `/chars`, `/img`, `/subscribe` 等快捷指令，并针对免费用户和 VIP 用户设计了 Freemium 商业化漏斗。

### 阶段二：多模态交互链路搭建 (Voice-to-Voice)
- **听觉感知与情绪嗅探**：不仅接入了 Deepgram ASR 实现极速的语音转文字，更通过“微提示词工程”，提取用户语音中的“语气与情绪”（如愤怒、失落、急促），并作为隐式上下文注入给大模型，让 AI 能做出情感一致的回复。
- **动态语音合成**：对接 Deepgram Aura / OpenAI TTS，根据剧情张力动态调整 AI 的发声情绪。
- **视觉生成**：基于 GLM-Image 设计了剧情驱动的图像生成工作流，AI 可根据当前场景自动生成并发送高质量的“场景照片”，极大推动了剧情发展。

### 阶段三：底层架构与高并发支持
- **异步通信桥接**：引入并深度改造了开源的 `ChatBridge` WebSocket 代理扩展，将底层单体 AI 引擎的流式事件转化为标准 HTTP 同步/异步响应，实现了极低延迟的逐字流式回复。
- **多租户隔离**：通过 SQLite 映射与动态 CSRF Token 获取机制，实现了多用户并发状态下的数据物理隔离和会话独立。

### 阶段四：长上下文记忆优化
- **智能滑动窗口**：结合摘要压缩机制，在不超出 API 限制的前提下，确保 AI 准确记住角色设定和关键历史剧情，彻底解决了长对话断片的问题。

## 4. 成果数据 (Results & Impact)
- **极致的响应速度**：通过全链路 `asyncio` 异步设计与流式传输，实现了毫秒级的语音转录与文本生成首字响应，交互体验如真人般顺畅。
- **高沉浸用户体验**：多模态（全双工语音+场景照）的引入，使得测试阶段用户的单日平均对话轮数与留存率显著提升。
- **系统高可用性**：成功突破了复杂代理环境下的长连接 SSL 断流问题，并实现了稳定的多租户并发隔离，保证了系统的高连通率与零数据串流。
